Explicación del Proyecto Web de Counter Strike

¿De qué trata el proyecto?

Esta página web fue creada como parte de una actividad académica, y tiene como objetivo mostrar información relacionada con el videojuego Counter Strike. En ella se presentan los diferentes modos de juego del título, además de su historia y versiones, permitiendo al usuario conocer las principales mecánicas y objetivos de cada uno.

¿Cómo se realizó la página?

La página fue elaborada únicamente con HTML, sin el uso de hojas de estilo CSS ni JavaScript, para enfocarnos exclusivamente en la estructura y el contenido. Se utilizaron varias etiquetas semánticas para organizar adecuadamente la información, haciendo que el sitio sea más legible y accesible tanto para los usuarios como para los motores de búsqueda.

El proyecto se compone de varios archivos HTML:

  • index.html: página principal o de inicio.
  • modos.html: donde se explican los diferentes modos de juego.
  • versiones.html: donde se explican las diferentes versiones del juego a lo largo de los años.
  • historia.html: aquí se habla acerca de la evolución del juego.
  • explicacion.html: este archivo, donde se detalla el proceso de creación y los conceptos usados.

Etiquetas semánticas utilizadas

<header>

Se utilizó para colocar el título principal y el menú de navegación en cada una de las páginas.

<nav>

Contiene los enlaces de navegación entre páginas, ayudando a organizar mejor la estructura del sitio.

<main>

Se empleó para englobar el contenido principal de cada página, diferenciándolo del resto del contenido repetido como el header y el footer.

<section>

Permite dividir el contenido en bloques temáticos, como los diferentes modos de juego o esta misma explicación.

<article>

Se utilizó en esta página para destacar definiciones o explicaciones individuales de conceptos.

<footer>

Se añadió al final de cada página para mostrar información complementaria como derechos de autor o nombre del proyecto.

Conclusión

Gracias al uso de HTML semántico, la página no solo es más organizada, sino que también facilita futuras modificaciones y mejora la accesibilidad. Aunque el diseño visual es muy básico por no usar estilos, la estructura está correctamente definida y es funcional para fines académicos y de práctica.
